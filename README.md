## About Me

Backend engineer currently working at **PDDN Inc** (Fremont, CA) while actively looking for my next full-time role. I spend most of my time writing Java and Spring Boot — building REST APIs, debugging request flows, thinking through DB design, and figuring out why something works in local but breaks in Docker.

I enjoy the parts of backend work that aren't glamorous: tracing down a slow query, cleaning up a messy service layer, reading logs to understand what actually happened. I'm not a flashy-project developer — I build things to understand them, and I write code I can come back to.

## Projects

### 🚆 [Train Ticket Booking System](https://github.com/oveshshaikh786/Train-ticket-booking)
A full-stack booking platform built with Spring Boot on the backend and React on the frontend, containerized with Docker. Built this to learn how to actually structure a real service — controller/service/repository layers, JWT auth, proper DB schema. The goal was to understand how a booking flow works end-to-end, not just make it run.

`Java` `Spring Boot` `React` `Docker` `JWT` `MySQL`

---

### 🐦 [Twitter Clone](https://github.com/oveshshaikh786/Twitter-Clone-Next.js-app)
Full-stack social platform with real-time updates via WebSockets. Built it to understand how real-time data flows work — optimistic UI on the frontend, event handling on the backend, and how to keep state consistent under concurrency. Next.js + MongoDB.

`Next.js` `React` `WebSockets` `MongoDB` `JWT`

---

### 🎓 [University Course Management](https://github.com/oveshshaikh786/university-course-management-java)
Java project focused on proper OOP architecture — abstraction, interfaces, inheritance, Factory/Observer/Singleton design patterns, 5-layer clean architecture. Built this to get a real grip on how large Java codebases are structured before working in one.

`Java` `OOP` `Design Patterns` `JUnit` `Maven`

---

### 📊 [Contoso Data Warehouse](https://github.com/oveshshaikh786/contoso-data-analysis-dw)
BI project built on top of the Contoso retail dataset. Star schema design, OLAP cube queries in MDX, 20+ business reports, complex aggregations over 1M+ records. Helped me understand how data moves from raw transactions to analytical reporting.

`SQL Server` `OLAP` `MDX` `Star Schema` `Power BI`
